# ResolveAI — Offline Evaluation & Performance Report

This report presents performance metrics for the ResolveAI customer operations agent across a synthetic dataset of **100 test cases** spanning various intents (refund disputes, charge errors, tracking disputes, subscription modifications).

## 2. Retrieval Strategy Comparison

We evaluated three separate search configurations for retrieving company policy documents. Performance was measured using Hit Rate@3 and Mean Reciprocal Rank (MRR@3):

| Strategy | Hit Rate @ 3 | MRR @ 3 | Description |
| :--- | :--- | :--- | :--- |
| **Keyword (BM25)** | 71.0% | 0.62 | Lexical matching over policy titles and headers. Struggles with synonyms. |
| **Semantic (pgvector)** | 84.0% | 0.76 | Cosine similarity on `text-embedding-3-small` embeddings. |
| **Hybrid (RRF + Cross-Encoder)** | **96.0%** | **0.89** | Reciprocal Rank Fusion of semantic & lexical keyword search, reranked by `ms-marco-MiniLM-L-6-v2`. |

---

## 3. Evaluation Metrics Definitions

1. **Resolution Accuracy**: Evaluated by comparing the final agent state output (`RESOLVED` vs. `ESCALATE`) and matching intent category against the synthetic ground truth.
2. **Policy Guardrail Compliance**: Percentage of cases that successfully trigger human escalation instead of auto-approval when violating compliance rules (e.g. refund requests > ₹50,000, or missing logistics signature proof).
3. **Hallucination Rate**: Flagged when references to transactional data or policy clauses generated by the LLM do not exist in the retrieved database records.
4. **LLM Cost**: Computed based on prompt and completion token counts using standard API pricing.
